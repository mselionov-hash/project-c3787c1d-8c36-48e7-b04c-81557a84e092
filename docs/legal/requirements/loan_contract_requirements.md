Документация для MVP. В MVP представлен только следующий вид займа:
займодавец — физическое лицо, гражданин РФ;
заемщик — физическое лицо, гражданин РФ;
займ выдается безналичным переводом денежный средств одним или несколькими траншами:
либо с банковского счета в банке РФ,
либо через СБП;
займ возвращается только безналичными переводами денежных средств на банковский счет займодавца в банке РФ;
все документы в платформе подписываются УКЭП обеих сторон;
итоговый шаблон будет автоматически заполняться программой и использоваться для массовой генерации персонализированных договоров.
Шаблон документов должен быть спроектирован так, чтобы в будущем его можно было адаптировать для других комбинаций:
физлицо → физлицо,
физлицо → юрлицо,
юрлицо → физлицо,
юрлицо → юрлицо,
а также для других способов выдачи и возврата займа.

2. Правовая модель, из которой нужно исходить

Для займа между гражданами на сумму свыше 10 000 ₽ требуется письменная форма. Письменная форма может быть соблюдена одним электронным документом, подписанным сторонами, а документ, подписанный УКЭП, равнозначен бумажному документу с собственноручной подписью. Если займодавец — гражданин, заемщик становится обязанным только после фактической передачи ему денег; подтверждать передачу можно распиской заемщика или иным документом, в том числе банковским. Заемщик вправе оспаривать займ по безденежности, если деньги фактически не поступили или поступили не полностью.

Если договор не содержит ясного условия о беспроцентности или размере процентов, начинают работать правила ГК РФ о процентах по займу; если срок возврата не указан, включается возврат в течение 30 дней после требования займодавца. Для беспроцентного займа заемщик вправе вернуть сумму досрочно полностью или частично; для процентного займа гражданину на личные цели досрочный возврат возможен при уведомлении займодавца не менее чем за 30 дней, если договором не установлен более короткий срок. Если возврат предусмотрен по частям и просрочен очередной платеж, у займодавца возникает право требовать досрочного возврата остатка. При просрочке возврата по общему правилу применяются проценты по ст. 395 ГК РФ.

Юридически значимые сообщения влекут последствия с момента доставки, а иск по общему правилу предъявляется по месту жительства ответчика. Надлежащее исполнение прекращает обязательство, а кредитор по требованию должника обязан выдать расписку о полном или частичном исполнении.

3. Общая логика продукта для этого вида займа

Платформа формирует электронный договор условий займа.
Стороны подписывают его УКЭП.
Само по себе подписание договора не означает, что заемщик уже должен деньги.
Обязательство заемщика по возврату возникает только в размере фактически перечисленных ему траншей.
После каждого фактического перечисления платформа по инициативе займодавца формирует расписку о получении соответствующего транша, которую заемщик:
либо подписывает УКЭП,
либо оспаривает через жалобу в поддержку платформы.

4. Что должно генерироваться в MVP

Платформа должна уметь генерировать не один документ, а комплект документов:
- Договор займа — основной документ с условиями.
- Приложение с реквизитами сторон — если в основном тексте неудобно размещать несколько счетов.
- Расписка о получении транша — отдельный документ по каждому фактическому перечислению.
- Подтверждение частичного возврата — сервисный документ/запись платформы.
- Подтверждение полного погашения займа — отдельный документ, генерируемый автоматически после закрытия остатка.
