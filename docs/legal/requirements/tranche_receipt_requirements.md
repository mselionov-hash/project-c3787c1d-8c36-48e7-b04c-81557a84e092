Техническое задание на шаблон расписки о получении транша по договору денежного займа
MVP | физлицо РФ → физлицо РФ | выдача безналично | возврат безналично | подпись УКЭП

Ключевой принцип: Расписка о получении транша — это узкий доказательственный документ, а не мини-договор. Она подтверждает конкретное поступление конкретной суммы по конкретному траншу, но не изменяет условия договора, не подменяет банковский документ и не должна содержать лишних признаний заемщика за пределами факта получения средств.

1. Назначение ТЗ

Настоящее ТЗ определяет требования к шаблону электронного документа «Расписка о получении транша» для того же вида займа, который уже описан в документации MVP и в шаблоне договора денежного займа. Будущий шаблон должен позволять платформе автоматически генерировать персонализированную расписку по каждому фактически перечисленному траншу без ручного редактирования текста пользователями.

2. Правовая модель расписки

При займодавце-гражданине заем является реальным: обязанность заемщика по возврату возникает только в размере фактически переданных денежных средств. Расписка должна подтверждать именно факт получения конкретного транша; не должна подменять банковское подтверждение перевода; не должна создавать долг раньше перевода; должна быть привязана к конкретному договору, конкретному траншу, конкретному платежному документу и снапшоту допустимых реквизитов.

3. Роль расписки в комплекте документов

Один транш — одна расписка. Один документ расписки не должен покрывать несколько отдельных переводов.

4. Статусы расписки

| Статус | Смысл |
|---|---|
| DRAFT | Проект сформирован, но еще не направлен заемщику на подпись |
| PENDING_SIGNATURE | Проект показан заемщику и ожидает действия |
| SIGNED | Заемщик подписал расписку УКЭП |
| DISPUTED | Заемщик не подписал расписку и инициировал спор / жалобу |
| VOID | Проект утратил актуальность и не должен использоваться |
